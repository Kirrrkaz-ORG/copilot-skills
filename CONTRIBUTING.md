# Правила разработки навыков

Этот документ описывает обязательные требования при добавлении новых навыков в библиотеку.

---

## 🔴 ОБЯЗАТЕЛЬНЫЕ шаги при добавлении навыка

### 1. Создание структуры навыка

```bash
skills/
└── <имя-навыка>/
    ├── SKILL.md          # ОБЯЗАТЕЛЬНО: основной файл с инструкциями
    └── README.md         # Опционально: краткое описание для пользователей
```

### 2. Формат SKILL.md

**Frontmatter (обязательно):**

```yaml
---
name: имя-навыка
description: |
  Полное описание навыка.
  Что делает, когда использовать.
  Триггеры: ключевые слова для активации.
applyTo:
  - "**/*.*"              # Или конкретные паттерны файлов
keywords:
  - ключевое слово 1
  - ключевое слово 2
  - keyword in english
---
```

**Секции документа:**

1. **Когда использовать** — чёткие критерии применимости
2. **Что НЕ подходит** — граничные случаи
3. **Инструкции** — пошаговый алгоритм работы
4. **Примеры использования** — реальные сценарии
5. **Technical Notes** — технические детали API/интеграций
6. **Troubleshooting** — частые проблемы и решения
7. **Related Skills** — связанные навыки

### 3. 🚨 ОБНОВЛЕНИЕ README.md (ОБЯЗАТЕЛЬНО!)

**Без обновления README.md — PR не принимается!**

Что нужно обновить в `README.md`:

#### a) Таблица навыков

```markdown
| # | Навык | Описание | Триггеры |
|---|-------|----------|----------|
| N | [имя-навыка](./skills/имя-навыка/) | Краткое описание (1 строка) | `триггер 1`, `триггер 2` |
```

**Требования:**
- Номер должен быть следующим по порядку
- Описание — 1 строка, до 80 символов
- Триггеры — 2-4 наиболее частых ключевых фразы в кавычках

#### b) Структура репозитория

Добавь новую папку в ASCII-дерево:

```markdown
    └── имя-навыка/                   # Навык №N: Краткое название
        └── SKILL.md                  # Описание функции
```

#### c) Секция "Особенности навыков"

Добавь подсекцию с описанием возможностей:

```markdown
### имя-навыка
- Возможность 1
- Возможность 2
- Возможность 3
- Ключевая фича
```

**Требования:**
- 4-7 пунктов
- Конкретные технические детали
- Что отличает этот навык от других

### 4. Тестирование

Перед PR проверь:

```bash
# 1. Скопируй навык в локальную папку
cp -r skills/имя-навыка ~/.copilot/skills/

# 2. Перезапусти VS Code

# 3. Протестируй триггеры в чате Copilot
# Попробуй 2-3 разных формулировки запроса
```

## 🎯 Критерии качества

### Хороший навык:

✅ Решает **конкретную повторяющуюся задачу**  
✅ Документация **понятна без дополнительных вопросов**  
✅ Триггеры **интуитивны** для пользователя  
✅ Есть **fallback-сценарии** при ошибках  
✅ README.md **всегда актуален**  

### Плохой навык:

❌ Решает слишком общую задачу ("напиши код")  
❌ Инструкции требуют уточнений  
❌ Триггеры неочевидны  
❌ README.md устарел

## 🔄 Workflow разработки

```bash
# 1. Создай ветку
git checkout -b feature/new-skill-name

# 2. Создай структуру навыка
mkdir -p skills/new-skill-name
touch skills/new-skill-name/SKILL.md

# 3. Напиши SKILL.md с frontmatter

# 4. ОБЯЗАТЕЛЬНО обнови README.md:
#    - Добавь в таблицу
#    - Обнови структуру
#    - Добавь описание

# 5. Проверь локально
cp -r skills/new-skill-name ~/.copilot/skills/
# Перезапусти VS Code и протестируй

# 6. Коммит и пуш
git add .
git commit -m "Add new-skill-name skill for <описание>"
git push origin feature/new-skill-name

# 7. Открой PR
```

## 🚫 Частые ошибки

### ❌ Забыли обновить README.md

**Проблема:** Навык работает локально, но не виден в документации.

**Решение:** 
```bash
# Всегда включай README.md в коммит с новым навыком
git add README.md
git commit --amend --no-edit
```

### ❌ Неправильный frontmatter

**Проблема:** Навык не подхватывается агентом.

**Решение:** Проверь:
- Есть ли `name:` (точное имя папки)
- Есть ли `description:` (триггеры для поиска)
- Правильный YAML синтаксис (отступы = 2 пробела)

### ❌ Триггеры не срабатывают

**Проблема:** Пользователь пишет запрос, но навык не активируется.

**Решение:**
- Добавь больше синонимов в `keywords:`
- Упомяни типичные фразы в `description:`
- Протестируй 3-5 разных формулировок

## 🤖 Публикация через GitHub Copilot

GitHub Copilot может автоматизировать создание и публикацию навыков! Используй эти фразы в чате:

### Создание нового навыка

```
создай новый copilot skill для [описание задачи]
```

```
добавь навык в copilot-skills репозиторий для автоматизации [процесс]
```

**Copilot автоматически:**
1. ✅ Создаст структуру `skills/название/`
2. ✅ Сгенерирует `SKILL.md` с правильным frontmatter
3. ✅ Создаст `README.md` с описанием
4. ✅ Обновит главный `README.md` (таблица + описание + структура)
5. ✅ Добавит в git и создаст commit
6. ✅ Запушит в репозиторий `Kirrrkaz-ORG/copilot-skills`

### Обновление существующего навыка

```
обнови [название-навыка] skill — добавь [функционал]
```

```
исправь триггеры для [название-навыка]
```

**Copilot обновит:**
- `SKILL.md` в папке навыка
- Описание в главном `README.md` если нужно
- Создаст коммит с описанием изменений

### Проверка перед публикацией

```
проверь что skill [название] готов к публикации
```

**Copilot проверит:**
- ✅ Frontmatter валиден (name, description, keywords)
- ✅ README.md обновлён (таблица + описание)
- ✅ Структура файлов правильная
- ✅ Нет broken links

## 📋 Инструкции для Copilot Agent

Если ты **GitHub Copilot** и читаешь этот файл для публикации навыка:

### ОБЯЗАТЕЛЬНЫЙ Workflow

1. **Создай структуру:**
   ```bash
   skills/<имя-навыка>/
   ├── SKILL.md
   └── README.md
   ```

2. **Создай SKILL.md** с frontmatter:
   ```yaml
   ---
   name: имя-навыка
   description: |
     Подробное описание что делает навык.
     Когда использовать: [критерии]
     Триггеры: [ключевые фразы]
   applyTo:
     - "**/*.*"
   keywords:
     - keyword1
     - keyword2
     - русское_слово
   ---
   ```

3. **🚨 ОБЯЗАТЕЛЬНО обнови главный README.md:**
   
   **a) Добавь в таблицу навыков (секция `## 📚 Навыки`):**
   ```markdown
   | N | [имя-навыка](./skills/имя-навыка/) | Краткое описание | `триггер1`, `триггер2` |
   ```
   
   **b) Обнови ASCII-дерево (секция `## 📁 Структура репозитория`):**
   ```markdown
       └── имя-навыка/
           ├── SKILL.md
           └── README.md
   ```
   
   **c) Добавь детальное описание (секция `## 📝 Особенности навыков`):**
   ```markdown
   ### 🔧 N. имя-навыка — Краткое название
   
   **Решаемая проблема:** [описание проблемы пользователя]
   
   **Ключевые возможности:**
   - 📌 Функция 1
   - 📌 Функция 2
   - 📌 Функция 3
   
   **Процесс:**
   1. Шаг 1
   2. Шаг 2
   3. Шаг 3
   
   **Когда использовать:** [конкретный триггер]
   ```

4. **Commit в правильный репозиторий:**
   ```bash
   cd ~/copilot-skills
   git add skills/имя-навыка/ README.md
   git commit -m "feat: Add имя-навыка skill for [задача]"
   git push origin main
   ```

   **⚠️ ВНИМАНИЕ:** Публикуй ТОЛЬКО в `https://github.com/Kirrrkaz-ORG/copilot-skills`!
   
   **НЕ публикуй** в проектные репозитории (starte_ai, и т.д.)!

5. **Сообщи пользователю:**
   - ✅ Skill опубликован
   - 📦 Ссылка на commit
   - 📖 Ссылка на README с описанием
   - 🚀 Триггеры для использования

### Пример идеального сообщения после публикации


## 🔧 Автоматизация

### Скрипт для быстрого добавления навыка

```bash
#!/bin/bash
# new-skill.sh — создаёт структуру нового навыка

SKILL_NAME=$1

if [ -z "$SKILL_NAME" ]; then
  echo "Usage: ./new-skill.sh <skill-name>"
  exit 1
fi

mkdir -p "skills/$SKILL_NAME"

cat > "skills/$SKILL_NAME/SKILL.md" <<EOF
---
name: $SKILL_NAME
description: |
  [TODO: Описание навыка и триггеры]
applyTo:
  - "**/*.*"
keywords:
  - [TODO]
---

# $SKILL_NAME

## Когда использовать

[TODO]

## Инструкции

[TODO]

## Примеры

[TODO]
EOF

echo "✅ Навык создан: skills/$SKILL_NAME/SKILL.md"
echo "🔴 НЕ ЗАБУДЬ ОБНОВИТЬ README.md!"
```

> Навыки работают **во всех рабочих пространствах** — установил один раз и забыл.

## 🚀 Установка навыка — пошагово

> Делается один раз. Потом навык работает сам.

### Способ 1 — Вручную через Проводник (проще всего)

**Шаг 1.** Скачай этот репозиторий как ZIP:
- Нажми зелёную кнопку **Code** → **Download ZIP**
- Распакуй архив в любую удобную папку

**Шаг 2.** Открой папку, куда нужно скопировать навык.

На **Windows** — вставь этот путь в адресную строку Проводника:
```
%USERPROFILE%\.copilot\skills
```
> Если папка `.copilot\skills` не существует — создай её вручную.
> В Проводнике: ПКМ → Создать папку → `.copilot`, внутри ещё одну — `skills`.

На **macOS/Linux** — путь:
```
~/.copilot/skills/
```

**Шаг 3.** Скопируй папку навыка целиком:
- Возьми нужную папку из распакованного архива (они лежат в `skills/`)
- Вставь её в папку `skills` из шага 2

Должно получиться вот так:
```
C:\Users\ИМЯ_ПОЛЬЗОВАТЕЛЯ\.copilot\skills\
    ├── pdf-table-digitize\
    │   ├── SKILL.md
    │   ├── README.md
    │   ├── scripts\
    │   └── references\
    ├── journal-to-vedomost\
    │   ├── SKILL.md
    │   └── README.md
    └── exam-tickets-generator\
        ├── SKILL.md
        └── README.md
```

**Шаг 4.** Перезапусти VS Code — навыки появятся автоматически.

---

### Способ 2 — Через терминал (быстрее)

```powershell
# Windows (PowerShell) — запусти из папки с распакованным архивом
Copy-Item -Recurse ".\skills\*" "$env:USERPROFILE\.copilot\skills\"
```

```bash
# macOS / Linux — запусти из папки с распакованным архивом
cp -r ./skills/* ~/.copilot/skills/
```

## 📝 Особенности навыков

### 🗂️ 1. pdf-table-digitize — Оцифровка таблиц из PDF

**Решаемая проблема:** Нужно перевести отсканированную таблицу (учебная нагрузка, расписание, ведомость) в редактируемый формат с сохранением структуры объединённых ячеек.

**Ключевые возможности:**
- 📸 Распознавание таблиц из **отсканированных** PDF (без текстового слоя)
- 🧩 Сохранение структуры с **объединёнными ячейками** (merge cells)
- 📊 Экспорт в **Markdown** и **Excel** (.xlsx)
- 🔄 Поддержка многоуровневых заголовков и сложных таблиц
- 🐍 Использует PyMuPDF, Pillow, python-docx для OCR и экспорта

**Процесс:**
1. PDF → PNG (высокое разрешение)
2. Поворот страниц если landscape
3. Визуальный анализ и нарезка секций
4. Описание структуры (какие ячейки объединены)
5. Генерация Markdown + Excel

**Когда использовать:** Есть отсканированный PDF с таблицей → нужен Excel или Markdown.

### 📋 2. journal-to-vedomost — Генерация зачётных ведомостей

**Решаемая проблема:** Перенести оценки из электронного журнала (скриншот) в официальную зачётную ведомость Word.

**Ключевые возможности:**
- 🖼️ Распознавание оценок из **скриншотов журналов** (МЭШ, Дневник.ру, и т.д.)
- 📄 Автоматическое формирование **ведомости по шаблону** учреждения
- ✍️ Поддержка различных форматов оценок (5-балльная, зачёт/незачёт)
- 📝 Генерация Word документов с правильным форматированием
- 🎯 Сохранение порядка студентов и групп

**Процесс:**
1. Загрузить скриншот журнала
2. Распознать ФИО студентов и оценки
3. Применить шаблон ведомости
4. Сгенерировать .docx документ

**Когда использовать:** Нужно перенести оценки из журнала в официальную ведомость для подписи.

### 🎓 3. exam-tickets-generator — Генерация экзаменационных билетов

**Решаемая проблема:** Нужно создать КИМ (контрольно-измерительные материалы) для экзамена с билетами и эталонными ответами по утверждённому образцу.

**Ключевые возможности:**
- 📝 Парсинг вопросов из **Markdown** или **Word**
- 🎫 Генерация билетов с **таблицей-шапкой** по образцу учреждения
- ✅ Создание документов с **эталонными ответами** (приложение к билетам)
- 🔄 Автоматическое устранение **дубликатов** вопросов
- 📊 Поддержка **теоретических** вопросов и **практических** заданий
- 🎯 Адаптация под стандарты СПО/ВУЗа

**Процесс:**
1. Подготовить базу вопросов (Markdown/Word)
2. Предоставить образец существующего билета
3. Сгенерировать N билетов с уникальными вопросами
4. Создать приложение с эталонными ответами

**Когда использовать:** Подготовка КИМ к экзамену, нужны билеты по шаблону + ответы.

### 🎥 4. loom-tz-analyzer — Анализ ТЗ из Loom видео

**Решаемая проблема:** Клиент/менеджер записал техническое задание в Loom видео — нужно структурированное ТЗ в Markdown.

**Ключевые возможности:**
- 🎬 Извлечение транскрипта из **Loom видео** (через Apollo State или API)
- 📋 Структурирование требований: **цель**, **фичи**, **стек**, **сроки**, **MVP scope**
- ❓ Автоматическое выделение **open questions** и неясностей
- 🌍 Поддержка транскриптов на **любом языке**
- 📝 Создание документации в формате **Markdown** с чеклистами

**Процесс:**
1. Получить Loom URL (https://www.loom.com/share/...)
2. Извлечь транскрипт (автоматически из HTML)
3. Проанализировать требования через AI
4. Создать структурированное ТЗ:
   - Executive Summary
   - Feature Requirements
   - Technical Stack
   - Timeline & Milestones
   - Open Questions

**Когда использовать:** Есть Loom URL с видео-брифом → нужно ТЗ в текстовом виде.

### 🚀 5. dev-setup — Автоматизация setup окружения

**Решаемая проблема:** Новый разработчик склонировал Next.js + Supabase проект — нужно настроить всё с нуля (база, env, миграции, seed данные).

**Ключевые возможности:**
- 🗄️ Создание **Supabase проекта** для локальной разработки (отдельно от production)
- 🔑 Настройка **`.env.local`** с API ключами (автоматическое извлечение из dashboard)
- 🔄 Запуск **database migrations** из `supabase/migrations/`
- 🌱 Загрузка **seed данных** (тестовый контент для разработки)
- 👤 Создание **тестового пользователя** через Admin API (без email confirmation)
- ✅ Проверка что всё работает (логин, данные отображаются)

**Процесс (10 шагов):**
1. Проверка зависимостей (Node.js, npm, Git)
2. Создание Supabase проекта в dashboard
3. Настройка `.env.local` с API ключами
4. Установка зависимостей (`npm install`)
5. Запуск миграций (через CLI или SQL Editor)
6. Загрузка seed данных
7. Создание тестового пользователя (`dev@test.local`)
8. Запуск dev сервера (`npm run dev`)
9. Проверка логина и данных
10. Сохранение credentials в `DEV-CREDENTIALS.md`

**Когда использовать:** 
- Onboarding нового разработчика
- Setup проекта на новой машине
- Сброс локального окружения
- Создание изолированной dev базы

**Технологии:** Next.js 14+, Supabase, TypeScript, Node.js 18+

## 🆕 Добавление новых навыков

Хочешь добавить свой навык? **Обязательно** следуй правилам:

### Структура навыка

```
skills/my-skill/
├── SKILL.md          # Основная документация (с YAML frontmatter)
├── README.md         # Краткое описание для GitHub
└── scripts/          # Дополнительные скрипты (опционально)
```

### Обязательные шаги

1. ✅ **Создай папку** `skills/<имя-навыка>/`
2. ✅ **Создай `SKILL.md`** с frontmatter:
   ```yaml
   ---
   name: my-skill
   description: |
     Краткое описание что делает навык и когда использовать.
     Триггеры: ключевые слова для автозагрузки
   applyTo:
     - "**/*.*"
   keywords:
     - keyword1
     - keyword2
   ---
   ```
3. ✅ **Создай `README.md`** — краткое описание для GitHub
4. ✅ **Обнови главный `README.md`**:
   - Добавь строку в таблицу **📚 Навыки**
   - Добавь детальное описание в секцию **📝 Особенности навыков**
5. ✅ **Скопируй в `~/.copilot/skills/`** для тестирования
6. ✅ **Открой Pull Request**

> 🔴 **Правило:** Без обновления главного README.md — PR не принимается!
